## Context

Currently, the `slash-agent` presents all proposed shell commands uniformly. Users are prompted with a generic `Confirm action: [y]es / [n]o / [e]dit / [c]omment ?` without any assessment of the safety risk or potential side effects of the command. This makes it easy for developers to accidentally run destructive or administrative commands (e.g. `rm -rf`, `sudo`). 

## Goals / Non-Goals

**Goals:**
- Extend the `execute_command` tool parameter signature to accept `risk_level` and `risk_description` generated by the LLM.
- Implement a Python-level safety guardrail that strictly overrides the risk classification for critical pattern matches.
- Print clear, color-coded risk levels (Safe, Low, Moderate, Critical) and explanations in the confirmation dialogue.
- Introduce a CLI flag `--unsafe-yes` to allow auto-confirming even `critical` actions.
- Restrict standard `-y` / `--yes` flag to only auto-confirm `safe`, `low`, or `moderate` risk commands.

**Non-Goals:**
- Deep semantic auditing of binary executions or external script contents.

## Decisions

### 1. Tool Parameters and Self-Assessment
We will update `execute_command` parameters:
```python
async def execute_command(command: str, risk_level: str = "low", risk_description: str = "") -> str:
```
The agent will fill in `risk_level` (`safe`, `low`, `moderate`, `critical`). The `risk_description` parameter is optional for `safe` and `low` risk levels, but MUST be provided for `moderate` and `critical` risk levels to explain the potential risk.


### 2. Python Guardrail (Override) List
If any of these patterns are matched (case-insensitively), `risk_level` is forced to `critical`:
- `rm -rf`
- `sudo `
- `mkfs`
- `dd `
- `chmod -R`
- `chown -R`
- `/dev/sd`
If a script run is detected (e.g., contains `./`, `sh `, `bash `, `python `, `make `, `npx `, `npm run `), the risk will be elevated to at least `moderate` (if not already higher).

### 3. Color Styling (ANSI)
In `prompt_user_confirmation`:
- **Safe**: `\033[1;32m[Risk: Safe]\033[0m` (Bold Green)
- **Low**: `\033[1;36m[Risk: Low]\033[0m` (Bold Cyan)
- **Moderate**: `\033[1;33m[Risk: Moderate]\033[0m` (Bold Yellow)
- **Critical**: `\033[1;31m[Risk: Critical]\033[0m` (Bold Red)

### 4. ARG/CLI Integration
In `slash_agent/main.py`:
- Add parser option `--unsafe-yes` (sets `session_state.unsafe_confirm = True`).
- If a command is `critical` and `session_state.unsafe_confirm` is `False`, prompt for confirmation even if `session_state.auto_confirm` is `True` (from `-y`).

## Risks / Trade-offs

- **Risk**: The user might use `--unsafe-yes` recklessly.
  - **Mitigation**: Standard documentation will strongly caution against default use of `--unsafe-yes`, emphasizing that it allows destructive commands to run without user oversight.
